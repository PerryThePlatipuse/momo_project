# Scaling laws — Herding coreset (отчёт для интерпретации)

Эксперимент: как качество классификации зависит от размера дистиллированного
датасета, отобранного методом **Herding**, для разных архитектур моделей и задач.

## Что это и как получено

- **Метод отбора:** Herding (greedy mean-matching в пространстве эмбеддингов).
  Отбор детерминированный, эмбеддинги для отбора считаются энкодером
  `bert-base-uncased` — один и тот же набор примеров отбирается для всех
  моделей-учеников (так корректно сравнивать архитектуры между собой).
- **Оценка:** для каждого датасета ученик обучается **с нуля** (предобученные
  веса + переинициализированная голова) на отобранном подмножестве и измеряется
  качество на полном валидационном/тестовом наборе задачи.
- **Метрика:** accuracy (для `qqp` — combined_score = mean(accuracy, F1), как в GLUE).
- **Размер датасета:** задаётся через **DPC** (примеров на класс).
  На графике по оси X отложено `K = DPC × число_классов`.

## Конфигурация прогона

| Параметр | Значение |
|---|---|
| Метод | herding |
| Задачи | sst2, mnli, qqp, ag_news |
| Модели-ученики | bert-base-uncased, bert-large-uncased, roberta-base, roberta-large, microsoft/deberta-v3-base, albert-base-v2, xlnet-base-cased |
| Сетка DPC | 1, 5, 10, 20, 50, 100, 500, 1000 |
| Повторов на точку | 3 переобучения (n_eval_per_dataset=3) |
| Шагов обучения | 200 |
| Batch size | 64 |
| Learning rate | 1e-4 (фиксированный для всех) |
| Точность | bf16 |
| Seed | 42 |

Всего: 7 моделей × 4 задачи × 8 значений DPC = **224 точки**, каждая усреднена
по 3 переобучениям. Числа в таблицах — `mean ± std` по этим 3 повторам.

## Файлы

- `scaling_laws_herding.png` — итоговый график (4 панели по задачам, линии по моделям, полоса = ±std).
- `summary_mean_std.csv` — сводная таблица: строки `(task, model)`, столбцы DPC, значения `mean ± std`. Для чтения людьми.
- `raw_results.csv` — сырые агрегаты (224 строки): mean, std, список scores по 3 повторам, время отбора и оценки.

## Ключевые выводы

- **Насыщение наступает рано.** Основной прирост качества идёт до DPC≈50–100,
  дальше кривые выходят на плато. На `ag_news` все модели упираются в ~0.91 уже
  при умеренных бюджетах.
- **`deberta-v3-base` — лучший почти везде** на больших бюджетах:
  mnli 0.834, qqp 0.811, sst2 0.937 (DPC=1000).
- **Задачи различаются по сложности дистилляции:** ag_news — лёгкая (высокое
  качество при малом K), mnli/qqp — заметно тяжелее (медленный рост, плато ниже).

## ВАЖНО: нестабильность крупных моделей

При фиксированном `lr=1e-4` крупные модели (`*-large`) иногда «срываются»: часть
из 3 повторов коллапсирует почти до случайного уровня. Это даёт большой `std` и
визуальные «провалы» линий на графике. Это **не ошибка сбора данных** (наоборот,
3 повтора это и вскрыли), а свойство обучения: lr высоковат для крупных моделей
на коротком прогоне в 200 шагов.

Наиболее заметные случаи (DPC=1000):
- `roberta-large`: sst2 0.639 ± 0.197, qqp 0.567 ± 0.162
- `bert-large-uncased`: mnli 0.601 ± 0.175

При интерпретации эти точки стоит трактовать с осторожностью (смотреть на std, а
не только на среднее). По решению команды конфиг оставлен как есть.

## Воспроведение

```bash
cd notebooks
python scaling_laws_run.py --prepare              # подготовить сырые датасеты
# 8 шардов по одному на GPU:
for i in 0 1 2 3 4 5 6 7; do
  CUDA_VISIBLE_DEVICES=$i python scaling_laws_run.py --shard $i --num-shards 8 &
done
python scaling_laws_run.py --collect              # собрать CSV + график
```
