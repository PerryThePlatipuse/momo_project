# Scaling laws — coreset-методы (random / k-centers), сравнение с herding

Дополняет `scaling_laws_herding_report/` (herding). Здесь добавлены **random** и
**k-centers**, чтобы получить сравнение трёх coreset-методов на одной сетке.

## Постановка

- Для каждого размера дистиллята **DPC** (примеров на класс; `K = DPC × число_классов`)
  ученик дообучается с нуля на отобранном подмножестве и оценивается на полном тесте.
- Конфиг (совпадает с herding-отчётом): 200 шагов, batch 64, lr 1e-4, bf16, seed 42,
  по 3 переобучения на точку (n_eval_per_dataset=3), метрика — accuracy.
- Отбор k-centers/herding — по эмбеддингам `bert-base-uncased` (один набор примеров
  для всех учеников). random — равномерная выборка по классам.

## Покрытие текущего прогона

## Выводы

1. **Дистилляция эффективна:** при DPC 1000 (~4000 примеров AG News = 0.3% данных)
   accuracy ~0.91, близко к полному обучению; при DPC 100 SST-2 — уже ~0.83–0.84.
2. **Умный отбор важен на малых бюджетах:** k-centers и herding заметно обгоняют random
   при DPC ≤ 20. Ярчайший разрыв — SST-2, DPC 5: 0.72 / 0.73 против **0.55** у random (+17 п.п.).
3. **На больших бюджетах разрыв исчезает:** при DPC ≥ 500 все три метода в пределах 1–2 п.п. —
   случайная выборка уже репрезентативна.
4. **k-centers ≈ herding** (оба отбирают по эмбеддингам): стабильного победителя между ними нет.
5. **Шум при DPC = 1:** на AG News random случайно оказался выше — при 1 примере/класс отбор вырожден.
